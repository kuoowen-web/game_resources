# Game Resources

## 專案目的

1. 研究理財與金錢相關的事情
2. 製作自用的理財小工具

## 隱私規則

- `player_stats.md` 和 `insights.md` 包含個人財務資料，不可 commit
- 寫入 `insights.md` 時不得包含可識別個人的財務數字，只記錄通用的研究結論

## 工作流程

- 當對話中產生關鍵 insight 或得出結論時，主動詢問使用者是否要記錄到 `insights.md`
- `insights.md` 格式：`日期 | 假設與 Insight | 信心程度`
- 個人財務背景資料記錄在 `player_stats.md`，討論中有更新時主動同步

## Skills 觸發情境

| Skill | 觸發情境 |
|-------|----------|
| **superpowers:brainstorming** | 設計新工具、新功能前，先探索需求與方向。**動手前必用** |
| **superpowers:writing-plans** | 有需求要拆成多步驟實作計畫時 |
| **superpowers:executing-plans** | 已有計畫，按計畫執行 |
| **superpowers:test-driven-development** | 理財計算邏輯不能算錯，先寫測試再寫實作 |
| **superpowers:systematic-debugging** | 遇到 bug 或非預期行為，系統化除錯 |
| **superpowers:verification-before-completion** | 宣稱完成前先跑驗證 |
| **simplify** | 程式碼寫完後檢查品質與效率 |
| **plan-discuss** | 「討論一下」「tradeoff」— CTO-CEO 技術討論模式 |
| **daily-recap** | 「今天做了什麼」「回顧」「daily recap」 |
| **cleanup-docs** | 「清理」「cleanup」— 清除暫存檔 |
| **skill-from-masters** | 討論 best practice、探討或採用新的研究及理財方法論時 |
| **update-docs** | 「update docs」「更新文件」「同步文件」— 掃描並更新專案文件 |
